정의
바이브코딩이란 정확히 무엇인가. 가장 단순한 정의에서 시작해 본질로 들어간다.
04
5.1 가장 단순한 정의
바이브코딩은 원하는 결과를 말로 설명하고
AI와 함께 작동하는 소프트웨어를 만들어가는 방식이다.
예를 들어 이렇게 말한다.
PROMPT
가족 일정 공유 앱을 만들어줘.
PROMPT
엑셀을 올리면 핵심 수치를 요약해주는 웹페이지 만들어줘.
PROMPT
구역모임 참석 여부를 받는 모바일 앱 만들어줘.
L01 · 01 정의
05
5.2 더 정확한 정의
단순 코드 생성이 아니라 대화 기반 반복 개발이다.
01
자연어 설명
코딩 언어가 아니라 일상 언어로 원하는 것을 말한다.
02
코드 생성
AI가 화면, 로직, 데이터 처리 코드를 만든다.
03
실행 결과 확인
사용자는 코드보다 결과 화면과 동작을 본다.
04
반복 수정
"이 부분 바꿔줘"라고 말하며 계속 개선한다.
말로 설명하고 → 결과를 보고 → 다시 말로 고친다. 이 사이클이 핵심이다.
L01 · 01 정의
06
5.3 결과 중심 개발
전통 코딩은 코드 중심, 바이브코딩은 결과 중심.
TRADITIONAL CODING
코드를 묻는다
  • 어떤 언어를 쓸 것인가?
  • 어떤 프레임워크를 쓸 것인가?
  • 어떤 구조로 짤 것인가?
  • 어떤 함수가 필요한가?
VIBE CODING
결과를 묻는다
  • 어떤 화면이 필요하지?
  • 사용자는 무엇을 해야 하지?
  • 버튼을 누르면 무슨 일이?
  • 결과가 마음에 드는가?

REASON
하면서 배운다
전통 학습
  1. 문법을 배운다
  2. 예제를 따라 한다

  4. 왜 안 되는지 묻는다
  5. 필요한 개념을 그때 배운다
→ 결과가 먼저, 개념은 나중에.
