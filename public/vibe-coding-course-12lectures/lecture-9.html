LECTURE 09
처음부터 DB로 가지 않는다
업무 데이터 실습의 첫 단계는 DB 연결이 아니라, 표 데이터를 안전하게 다루는 것입니다.
  • DB 설계는 어렵고, 사내 접근 권한도 필요하다
  • 데이터 정합성, 보안, 운영 책임이 즉시 따라온다
  • 비개발자 첫 실습은 복사·붙여넣기가 가장 안전하다
  • 엑셀 붙여넣기는 거의 모든 직군이 이해한다

LECTURE 09
붙여넣은 표를 읽는 방식
코드 내부에서는 표가 배열과 객체로 바뀝니다. 개념만 이해하면 됩니다.
단계설명
1. 줄 나누기줄바꿈 기준으로 행을 나눈다
2. 헤더 읽기첫 줄을 열 이름으로 본다
3. 칸 나누기탭 또는 쉼표 기준으로 값을 나눈다
4. 숫자 변환조회, 승인, 실행을 숫자로 바꾼다
5. 배열 생성각 행을 객체 목록으로 정리한다

PROMPT
Codex 첫 지시문
4강과 5강에서 배운 것처럼, 목적·입력·처리·출력·제약을 함께 전달합니다.
단일 HTML 파일로 엑셀 붙여넣기 보고 요약기를 만들어라. 요구사항: 1. 큰 텍스트 입력창에 TSV 또는 CSV 데이터를 붙여넣는다. 2. 첫 줄은 헤더로 처리한다. 3. 조회, 승인, 실행 열을 숫자로 계산한다. 4. 총 조회, 총 승인, 총 실행, 승인율, 실행률을 보여준다. 5. 채널별 실행 건수와 실행률을 표로 보여준다. 6. 계산 결과를 바탕으로 보고 문구를 생성한다. 7. 보고 문구 복사 버튼과 초기화 버튼을 만든다. 8. 서버, DB, 외부 API는 사용하지 않는다. 9. 모바일에서도 읽기 쉽게 만든다. 우선 1차 초안을 만들어라.

LECTURE 09
만들어진 뒤 먼저 볼 것
초안이 나오면 디자인보다 흐름과 오류 대응부터 확인합니다.
  • 붙여넣기 입력창이 충분히 큰가
  • 샘플 데이터가 없어도 사용법을 이해할 수 있는가
  • 계산 결과가 표와 문장으로 함께 보이는가
  • 복사 버튼이 실제로 동작하는가
  • 오류가 났을 때 사용자가 무엇을 고쳐야 하는지 알 수 있는가

LECTURE 09
데이터 보안 원칙
Part 3의 모든 실습은 보안 경계를 먼저 정하고 진행합니다.
  • 실습에는 교육용 샘플 데이터만 사용한다
  • 고객명, 연락처, 계좌번호, 식별번호는 넣지 않는다
  • 내부 실적 원자료는 외부 서비스에 붙여넣지 않는다
  • 저장 기능은 만들지 않는다
  • 필요하면 마스킹된 데이터로 바꾼다

LECTURE 09
흔한 함정
데이터 실습의 실패는 대부분 범위를 크게 잡거나 검증을 생략할 때 생깁니다.
  • 샘플 데이터 없이 바로 만들기 시작한다
  • 헤더 이름을 매번 바꿔 코드가 깨진다
  • 모든 지표를 한 번에 넣으려 한다
  • 계산 결과 검증 없이 보고 문구를 믿는다
  • 실제 회사 데이터를 외부 AI에 붙여넣는다
