LECTURE 05
왜 화면이 중요한가
  • 말로 하면 각자가 서로 다른 장면을 상상한다
  • 화면은 현업, 기획, 개발 사이의 해석 차이를 줄인다
  • 화면을 보면 누락된 기능과 불필요한 기능이 드러난다
  • 개발 전에 사용자 흐름을 싸게 검증할 수 있다

LECTURE 05
사용자 흐름
단계사용자 행동화면이 해야 할 일
1. 진입도구를 연다목적을 즉시 이해시킨다
2. 입력아이디어나 메모를 넣는다입력 예시와 안내를 제공한다
3. 선택결과 유형을 고른다선택지를 과도하게 늘리지 않는다
4. 실행생성 버튼을 누른다결과가 생성되는 위치를 명확히 한다
5. 활용결과를 복사한다복사·초기화·다시 작성이 가능해야 한다

LECTURE 05
좋은 화면 프롬프트
나는 Working Backwards 1페이지 생성기라는 웹 도구의 첫 화면 목업을 만들고 싶다. 이 도구는 비개발자 회사원이 자기 아이디어를 입력하면 보도자료 문장, KPI, 화면 설계 프롬프트, Codex 구현 프롬프트를 정리해주는 도구다. 첫 화면에는 아이디어 입력 영역, 생성 버튼, 결과 영역, 복사 버튼, 민감정보 입력 금지 안내가 필요하다. 디자인은 회사 업무용 SaaS 도구처럼 차분하고 명확해야 한다. 이번에는 실제 저장 기능이나 로그인은 만들지 말고, 화면 구조와 사용자 흐름이 보이는 목업으로 만들어라.

LECTURE 05
화면 설계 프롬프트 템플릿
나는 [도구 이름]의 첫 화면 목업을 만들고 싶다. 이 도구는 [사용자]가 [상황]에서 [문제]를 해결하도록 돕는다. 핵심 가치는 [가치]이고, 성공 기준은 [KPI]이다. 첫 화면에는 다음 요소가 필요하다. 1. 제목 2. 설명 문구 3. 입력 영역 4. 선택 옵션 5. 실행 버튼 6. 결과 표시 영역 7. 복사 또는 공유 기능 8. 주의사항 디자인은 [분위기]로 만들고, 이번에는 화면 구조와 사용자 흐름이 보이게 만들어라.

LECTURE 05
ChatGPT 또는 Lovable에 넣기
  • 먼저 화면 구조가 보이는 1차 목업을 요청한다
  • 실제 기능 구현보다 사용자 흐름을 우선 확인한다
  • 예쁜 색보다 정보 순서와 문구가 맞는지 본다
  • 생성된 결과를 보고 빠르게 한 번 수정한다
5강의 목표는 완성 화면이 아니라, 구현 기준이 되는 첫 화면이다.
