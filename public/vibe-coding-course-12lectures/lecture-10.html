LECTURE 10
API Key를 다룰 때의 원칙
이 슬라이드는 실습보다 경계 설정이 목적입니다.
  • HTML 파일 안에 API Key를 직접 넣지 않는다
  • GitHub에 키를 올리지 않는다
  • 브라우저에서 키가 노출되는 구조를 피한다
  • 비용과 호출량을 통제한다
  • 회사 데이터 전송 가능 여부를 먼저 확인한다

LECTURE 10
AI가 자주 실패하는 지점
AI 결과는 반드시 검토해야 합니다.
  • 없는 원인을 만들어낸다
  • 숫자를 부정확하게 옮긴다
  • 보고 목적과 맞지 않는 장문의 설명을 한다
  • 리스크와 사실을 구분하지 못한다
  • 사용자가 원하는 톤과 다르게 쓴다

PROMPT
Codex 수정 지시문
오늘 구현의 기본은 API 없는 안전 버전입니다.
기존 엑셀 붙여넣기 보고 요약기에 AI 보고 프롬프트 생성 기능을 추가해라. 추가 요구사항: 1. 보고 목적 선택 드롭다운을 만든다. - 상사 보고 - 팀 공유 - 임원 요약 - 리스크 검토 2. 계산 결과를 기반으로 AI에게 보낼 프롬프트를 생성한다. 3. 프롬프트에는 역할, 업무 맥락, 집계 결과, 요청사항, 제약조건을 포함한다. 4. 생성된 프롬프트를 별도 영역에 보여준다. 5. 프롬프트 복사 버튼을 만든다. 6. 외부 API 호출은 하지 않는다. 7. 민감정보 입력 금지 안내를 화면 상단과 프롬프트 영역에 표시한다. 기존 기능은 유지하고, 먼저 안전 버전으로 구현해라.
