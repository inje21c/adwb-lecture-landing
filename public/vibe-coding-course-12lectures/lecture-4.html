WHY NOT JUST TODO?

L04 · 프로젝트
15
FIRST CODEX PROMPT
2강의 원칙을 그대로 적용한다.
나는 비개발자도 자기 아이디어를 Working Backwards 방식으로 정리할 수 있는 로컬 웹 도구를 만들고 싶다. 도구 이름은 "Working Backwards 1페이지 생성기"다. 사용자는 회사 임직원이며, 각자 만들고 싶은 업무 도구나 서비스 아이디어가 있다. 화면에는 다음 입력 항목이 있어야 한다. 1. 만들고 싶은 것 2. 대상 사용자 3. 현재 불편 4. 기대 가치 5. 성공 KPI 6. 제약 조건 생성 버튼을 누르면 보도자료 문장, KPI 요약, 첫 화면 구성, 화면 생성 프롬프트, Codex 구현 프롬프트가 결과로 보여야 한다. 서버, DB, 로그인, 외부 API는 사용하지 않는다. HTML, CSS, JavaScript를 하나의 파일에 포함해 1차 초안을 만들어라.

L04 · 실행
18
FIRST FEEDBACK
처음에는 한 곳만 짚어 고친다.
전체적으로 더 좋게 만들어줘.
범위가 넓어져서 의도와 다른 변경이 생긴다.
결과 영역의 '화면 생성 프롬프트'가 너무 짧다. 사용자, 첫 화면 구성, 버튼, 결과 상태가 포함되도록 더 구체적으로 만들어줘. 단, 입력 항목은 늘리지 마라.
L04 · 피드백
19
ADD ONE USEFUL FEATURE
복사 버튼은 실무 도구의 핵심이다.
결과가 화면에만 있으면 데모다.
결과를 복사해서 메신저, 문서, Codex에 붙여넣을 수 있으면 도구가 된다.
각 결과 카드마다 '복사' 버튼을 추가해줘. 사용자가 버튼을 누르면 해당 결과 텍스트만 클립보드에 복사되게 해줘. 복사 후에는 버튼 문구를 잠시 '복사됨'으로 바꿔줘.
L04 · 기능 추가
20
PERSONAL APPLICATION
이제 각자의 아이디어를 넣는다.
입력 칸작성 예시
만들고 싶은 것회의 메모를 보고 문구로 바꾸는 도구
대상 사용자회의 후 상사나 팀에 내용을 공유해야 하는 회사원
현재 불편산만한 메모를 정리하는 데 시간이 오래 걸림
기대 가치보고 작성 시간을 줄이고 공유 속도를 높임
성공 KPI작성 시간 5분 → 1분, 재작성 횟수 2회 → 0~1회
제약 조건서버 없음, 개인정보 저장 없음, 로컬 실행
